Startloch-Geometrien

 

Wird eine Feature-Erkennung ausgeführt, erkennt OPTICAM nicht nur erodierbare Geometrien, sondern legt auch die Position der Startlöcher fest. Dabei wird die Position der Startlöcher nach einem intern festgelegtem Regelwerk bestimmt.

Mit Hilfe des Dialogbereiches Startloch-Geometrien, können Sie das interne Regelwerk außer Kraft setzen und die Position der Startlöcher durch Vorgabe von Startloch-Geometrien (Zylindrische Geometrien vom Typ "Oberfläche") selbst bestimmen.

Im Dialogbereich Startloch-Geometrien können Sie einstellen, wie das System vorhandene Startlochgeometrien interpretiert:

 

Startloch-Geometrien ignorieren

Alle vorhandenen Startloch-Geometrien werden ignoriert und es gilt das interne Regelwerk für die Positionierung der Startlöcher.

Das folgende Bild zeigt, dass die Feature-Erkennung alle existierenden Startlochgeometrien ignoriert und für alle Feature das interne Regelwerk für die Positionierung der Startlöcher verwendet.

 

 

Existierende Startloch-Geometrien verwenden

Existieren Startloch-Geometrien, werden diese erkannt und für die Positionierung der Startlöcher verwendet.

Für Features ohne vorhandenen Startloch-Geometrien gilt das interne Regelwerk für die Positionierung der Startlöcher.
Das folgende Bild zeigt, dass für die beiden Dreiecke das interne Regelwerk für die Positionierung der Startlöcher verwendet wird, da für diese Features keine Startlochgeometrien existieren.

 

 

Features ohne Startloch-Geometrien ignorieren

Legt fest, dass Bearbeitungen nur dann erzeugt werden, wenn für ein Feature auch eine Startloch-Geometrie existiert.
Das folgende Bild zeigt, dass nur die beiden Dreiecke unbearbeitet bleiben, da für diese Feature keine Startlochgeometrien existieren.

 

 

Startloch-Geometrien auch für Endposition verwenden

Ist diese Option aktiv und verfügt ein Feature über eine Startloch-Geometrie, dann fährt der Draht nach jedem Schnitt zurück in die Mitte der Startloch-Geometrie.  

Das folgende Bild zeigt links die Situation OHNE Aktivierung der Option Startloch-Geometrien auch für Endposition verwenden und rechts mit.

 

 

Mehrfach-Segmente erzeugen wenn möglich

Ist diese Option aktiv und verfügt ein Feature über mehrere Startloch-Geometrien, dann werden diese automatisch erkannt und das Feature in mehrere Segmente (Teilbereiche) aufgeteilt.

Das folgende Bild zeigt links die Wirkung der Option Mehrfach-Segmente erzeugen, da die drei Startloch-Geometrien erkannt und berücksichtigt werden.

Das rechte Bild zeigt die gleiche Geometrie, aber die drei Startlochgeometrien bleiben unberücksichtigt, da die Option Mehrfach-Segmente erzeugen falls möglich nicht aktiv ist.

 

 

Erkenne Startloch-Geometrien nach Abstand

Es kann passieren, dass das System Startloch-Geometrien nicht eindeutig zu einem Feature zuweisen kann.
Die Angabe eines Abstandwertes kann jedoch helfen das Problem zu lösen.

 

Das folgende Bild oberhalb zeigt 3 Stempel mit jeweils zwei Startloch-Geometrien, deren Abstand zur Schneidkontur 5 mm beträgt.
Bei Ausführung der Automatischen Feature-Erkennung war der Parameter Erkenne Startloch-Geometrien nach Abstand jedoch auf 10 mm eingestellt.
Dies führt dazu, dass die Startloch-Geometrien falsch erkannt werden.

            

Um das Problem zu lösen, stellt man den Parameter Erkenne Startloch-Geometrien nach Abstand auf 5mm ein. Auf diese Weise werden die Startloch-Geometrien korrekt erkannt.
